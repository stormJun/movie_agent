# 02-核心机制

> **四层渐进式学习结构**
> **总阅读时间**：4-8 小时（根据深入程度）
> **难度等级**：⭐⭐ 到 ⭐⭐⭐⭐

## 📋 四层结构说明

本章采用**四层渐进式学习结构**，帮助您从整体到细节、从概览到深入理解系统。

### 第一层：整体架构（⭐⭐ 30分钟）

快速理解系统"是什么"，适合所有读者。

- [系统架构总览](./01-整体架构/系统架构总览.md) - 组件关系、核心模块
- [数据流总览](./01-整体架构/数据流总览.md) - 从文档到回答的完整流程

### 第二层：核心子系统（⭐⭐⭐ 每篇30-60分钟）

深入理解各子系统"如何工作"，适合架构师和开发者。

- [Agent 系统](./02-核心子系统/Agent系统.md) - BaseAgent、5种Agent、LangGraph编排
- [知识图谱构建](./02-核心子系统/知识图谱构建.md) - 实体提取、关系抽取、图谱构建
- [搜索引擎](./02-核心子系统/搜索引擎.md) - Local/Global/Hybrid/DeepResearch
- [社区检测](./02-核心子系统/社区检测.md) - Leiden/SLLPA 算法
- [评估系统](./02-核心子系统/评估系统.md) - 20+ 评估指标

### 第三层：关键特性（⭐⭐⭐ 每篇30-45分钟）

突出差异化特性"优势在哪里"，适合架构师和技术决策者。

- [多 Agent 协作](./03-关键特性/多Agent协作.md) - Plan-Execute-Report 架构
- [实体消歧和对齐](./03-关键特性/实体消歧和对齐.md) - 字符串召回+向量重排+NIL检测
- [增量更新机制](./03-关键特性/增量更新机制.md) - 文档变更检测、冲突解决
- [Chain of Exploration](./03-关键特性/Chain-of-Exploration.md) - 图谱上的多步探索
- [流式响应](./03-关键特性/流式响应.md) - LLM 流式生成机制

### 第四层：深入理解（⭐⭐⭐⭐ 每篇60-90分钟）

源码级深度"如何实现"，适合二次开发者和贡献者。

- [存储模型详解](./04-深入理解/存储模型详解.md) - Neo4j 图结构、向量索引
- [代码走读](./04-深入理解/代码走读.md) - 核心类解析、关键函数
- [性能调优](./04-深入理解/性能调优.md) - 瓶颈分析、优化策略

## 🎯 学习路径建议

### 路径 1：快速理解（1小时）
第一层全部 → 完成

### 路径 2：深入理解（4小时）
第一层 → 第二层（选择感兴趣的子系统） → 第三层（选择关键特性）

### 路径 3：精通系统（8小时）
第一层 → 第二层全部 → 第三层全部 → 第四层（按需）
