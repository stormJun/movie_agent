## 文档内容要求

### 所有文档必须包含

1. ✅ **元信息块**
   ```markdown
   > **目标读者**：明确的读者群体
   > **阅读时间**：预计阅读时间
   > **前置知识**：需要提前了解的内容
   > **难度等级**：⭐ 到 ⭐⭐⭐⭐
   ```

2. ✅ **本文大纲** - 清晰的章节导航

3. ✅ **代码示例** - 技术文档至少 5-10 个代码示例

4. ✅ **流程图/架构图** - 使用 Mermaid 绘制（适用时）

5. ✅ **对比表格** - 特性对比、选项对比（适用时）

6. ✅ **相关文档链接** - 文档间的导航

7. ✅ **更新日志** - 记录文档变更

### 质量标准

- **准确性**: 技术内容准确、代码可运行
- **完整性**: 覆盖主题的所有重要方面
- **清晰性**: 表达清楚、逻辑连贯
- **实用性**: 提供可操作指导、真实案例
